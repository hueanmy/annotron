# Annotron Voice Script

## Version 1: Short Introduction (30 seconds)

```
Annotron is a local, browser-based review editor for AI-generated artifacts.

When your coding agent creates HTML plans, diagrams, or mockups, giving feedback 
is usually clumsy — you have to describe what you see in words.

With annotron, you simply click. Click any element or text to comment. The agent 
gets precise, structured feedback and applies changes in seconds. Watch the agent 
work live, see updates instantly, and repeat until you're happy.

It's loop engineering for artifacts — point at exactly what to change, and watch 
it happen in real time.

Annotron: feedback made precise.
```

## Version 2: Detailed Feature Tour (60 seconds)

```
Meet Annotron — a browser-based review editor for agent-generated artifacts.

Imagine your AI coding agent generates a 50-page architecture document. Instead 
of describing changes in paragraphs, just open it in annotron and click.

Click any text or element to comment. Your feedback carries the exact CSS selector, 
the selected text, and your note. The agent reads this structured feedback and 
applies changes immediately.

Watch the agent work live in the sidebar — see every Read, Edit, and Bash command 
stream in real time. The file updates as it changes, and your browser reloads with 
scroll position preserved.

Key features:

Point-and-click annotations — no prose, no ambiguity. Comment on any element or 
text selection.

Markdown mode with diagrams — render Mermaid flowcharts, sequence diagrams, and 
more as interactive SVG.

Outline navigation — for long documents, an auto-generated outline sidebar lets 
you jump between sections instantly.

Auto-apply feedback loop — structured feedback → agent applies changes → live 
preview update. No manual polling. No context switching.

History and threads — browse past feedback rounds and annotation conversations.

Available on VS Code, npm, and OpenVSX.

Annotron: from unclear feedback to precise edits in seconds.
```

## Tips for Recording

1. **Tone**: Professional, clear, and confident — not overly fast
2. **Pacing**: Pause after key points (especially feature names)
3. **Emphasis**: Stress "precise," "structured," "instant," and "loop engineering"
4. **Audio**: Use a clear microphone, minimal background noise
5. **Sync with GIF**: Time narration to match the demo GIF (3:56 duration)
   - Intro (0-5s): Set the problem
   - Feature walkthrough (5-45s): Outline, annotations, Markdown
   - Agent loop (45-90s): Live feedback, agent activity, live updates
   - Outro (90-120s): Summary + call to action
